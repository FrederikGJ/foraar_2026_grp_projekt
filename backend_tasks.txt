Pakke: dk.bilbase.backend


[FÆRDIG] Trin 1 — Maven skelet + DB-forbindelse
   - pom.xml (Spring Boot 3.5.3, Java 21, web/JPA/validation/MySQL)
   - Application.java
   - application.yml + application-dev.yml (MySQL localhost:3307)
   - .gitignore
   - Verificeret: app booter, HikariCP connecter til MySQL 8.0.45


[FÆRDIG] Trin 2 — Lookup-entities + repositories
   - domain/Role, Region, FuelType, Brand, Model
   - repository/RoleRepository (+ findByName), RegionRepository,
     FuelTypeRepository, BrandRepository, ModelRepository (+ findByBrandId)
   - Verificeret: ddl-auto: validate accepterer alle 5 mappings


[FÆRDIG] Trin 3 — AppUser + Spring Security + JWT auth
   - AppUser entity + AppUserRepository (+ existsByUsername/Email, dummy-fix queries)
   - AppUserPrincipal (custom UserDetails) + AppUserDetailsService
   - JwtService (HS512, jjwt 0.12.6) + JwtAuthenticationFilter
   - SecurityConfig (stateless, BCrypt, /api/auth/** public, lookups public)
   - AuthController: POST /register, /login, /logout + GET /me
   - DevDataInitializer (@Profile dev) — UPDATE'er 100 seed-users til bcrypt(password123)
   - Verificeret: login admin/dealer1, /me protected, forkert password -> 401
   - TODO trin 9: 403 -> 401 via AuthenticationEntryPoint


[FÆRDIG] Trin 4 — Car + CarListing + CarSale entities
   - domain/Address (+ Region @ManyToOne)
   - domain/Car (+ Model, FuelType @ManyToOne; BigDecimal price)
   - domain/CarListing (UDEN is_sold-kolonne; @OneToOne(mappedBy) CarSale; @Transient isSold())
   - domain/CarSale (UNIQUE car_listing_id, FK ON DELETE RESTRICT)
   - repository/AddressRepository, CarRepository,
     CarListingRepository (+ findBySellerId),
     CarSaleRepository (+ findByCarListingId, existsByCarListingId)
   - Verificeret: 10 JPA repos, ddl-auto: validate grøn


[MANGLER] Trin 5 — Listings CRUD + sale-endpoint
   - DTO'er (request + response)
   - ListingService med ownership-tjek
   - ListingController:
       GET    /api/listings           (filter, sort, paging)
       GET    /api/listings/{id}
       POST   /api/listings           (DEALER+ADMIN, @Transactional)
       PUT    /api/listings/{id}      (owner eller ADMIN)
       DELETE /api/listings/{id}      (owner eller ADMIN, 409 hvis salg findes)
       POST   /api/listings/{id}/sale (CUSTOMER+ADMIN)
   - Erstatter stored procedure create_car_listing med Java @Transactional


[MANGLER] Trin 6 — View-entities → read-only endpoints
   - @Immutable view-entities:
       ActiveListingView  → /api/listings/active
       CarDetailsView     → /api/cars
       UserMessageView    → bruges af /api/messages
       UserFavoriteView   → bruges af /api/favorites
   - Read-only repositories


[MANGLER] Trin 7 — Favorite + Message endpoints
   - Favorite entity + repository
   - Message entity + repository
   - FavoriteService (pre-validate: listing ikke solgt — trigger backup)
   - MessageService (pre-validate: sender != receiver, listing ikke solgt)
   - FavoriteController:
       GET    /api/favorites          (egne, via UserFavoriteView)
       POST   /api/favorites
       DELETE /api/favorites/{id}
   - MessageController:
       GET    /api/messages/inbox
       GET    /api/messages/outbox
       POST   /api/messages
